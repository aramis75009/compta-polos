# Workflow Git — Mac ↔️ Windows

## Avant de commencer à coder (sur n'importe quelle machine)
git pull

## Après chaque session de vibe coding (avant de fermer/changer de machine)
git add -A
git commit -m "description courte des changements"
git push

## Règle d'or
Ne jamais coder sur les deux machines en même temps sans pull/push entre les deux.
Toujours pull avant de commencer, toujours push avant de partir.

## Vercel
Le site se redéploie automatiquement à chaque push sur la branche main, 
peu importe la machine d'origine.
